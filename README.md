1 概述
Guru Agent OS 是一个用于提升企业整体执行效率的 Web 端业务系统。
在形态上，它是一个 Web App；
在能力组织上，它由多个可独立使用的 AI Agent 组成；
在协作机制上，不同 Agent 通过统一的「Card」进行上下文交互，从而减少重复判断与信息断层。
每个 Agent 对应一类明确的业务任务（如创意洞察、素材生成、投放执行、数据分析等），
用户可以像使用专用工具一样进入某一个 Agent，快速完成当前阶段的工作，而无需在多个系统间切换。
系统中产生的关键结论与经验，会被沉淀为 Card。
Card 作为通用的上下文载体，可以被保存、收藏，并在进入其他 Agent 时直接复用，
使已有判断无需反复重来。
通过 Agent 的分工执行与 Card 的跨 Agent 流转，
Guru Agent OS 将原本依赖个人记忆和人工传递的信息，转化为可持续复用的系统资产，
从而缩短决策链路，减少重复劳动，提升团队在人员规模不变情况下的整体执行效率。

2 UA 团队具体用例
在 UA 团队中，Guru Agent OS 的核心作用是提升整体执行效率，而不仅是单点岗位提效。
传统 UA 工作中，创意、素材、投放和数据分析分别由不同角色完成，信息通过口头、文档或表格传递，决策链路长、重复判断多，导致有效经验难以被快速复用。
Guru Agent OS 将上述流程拆解为多个独立 Agent，并通过统一的 Card 机制打通信息流转，使关键结论能够被系统性复用。
具体而言：
- 创意、素材、投放和数据分析分别在对应 Agent 中完成，减少跨工具、跨角色沟通成本；
- 各阶段产生的有效结论（如高效创意方向、已验证素材特征、明确的数据判断）会被自动或手动沉淀为 Card；
- Card 可被收藏，并在进入其他 Agent 时直接作为上下文使用，避免重复分析与重复决策。
在该模式下，数据分析阶段产出的结论可以被即时用于指导下一轮创意与素材生产，从而显著缩短“发现有效策略 → 实际执行”的时间。
通过 Card 在 Agent 之间的持续流转，UA 团队的有效经验能够被快速沉淀和放大，使团队在人员规模不变的情况下，提高出量速度与执行稳定性。

3 Roadmap
本项目采用阶段性交付与验收方式推进，每一阶段均产出明确、可验证的成果，用于支持阶段性投入与评估。

阶段一｜可展示版本（1.7 前）
目标：验证产品形态与核心交互是否成立
交付内容：
- Web App 基础框架完成，可正常访问与操作
- 至少 1 个 Agent 的可用页面（展示 Agent 独立功能形态）
- Card 作为独立信息单元的基础形态可展示（创建 / 展示）
- 能完整演示「进入 Agent → 生成结果 → 形成 Card」的基本流程

阶段二｜初步可用版本（2.14 前）
目标：验证 Agent + Card 在真实业务中的可用性
交付内容：
- 创意热点 Agent 与素材生成 Agent 上线
- Card 支持收藏、查看，并可作为上下文带入另一个 Agent
- 可在系统内完成一次完整的「创意 → 素材生成」流程
- UA 团队可在系统内完成创意与素材生成任务
- Card 能被实际用于减少重复输入与重复判断

阶段三｜决策支持版本（3.30 前）
目标：验证系统对业务决策效率的提升价值
交付内容：
- 策略生成 Agent 与数据分析 Agent 上线
- 数据分析结果可自动生成分析结论 Card
- 分析 Card 可被用于指导下一轮创意或素材生成
- 系统内形成「执行 → 数据 → 结论 → 再执行」的闭环
- UA 成员可直接基于系统结论进行下一步决策

阶段四｜完整闭环版本（4.30 前）
目标：实现可持续运转的业务执行系统
交付内容：
- 全流程 Agent 打通，统一 Context 机制生效
- Card 在不同 Agent 间的流转路径清晰、稳定
- 核心业务经验可被持续沉淀并复用
- 系统可支持 UA 团队持续使用，而非一次性试验
- 团队在人员规模不变的情况下，决策与执行效率有明显提升
